# Docker Multi-Environment Configuration Design

**Date**: 2026-03-16
**Status**: Approved
**Author**: AI Assistant + User

## Summary

Redesign Docker Compose configuration from 14+ fragmented files into a clean 4-environment system: **dev** (all local), **integration** (local code + remote services), **staging** (meepleai.app), and **prod** (template for future deployment).

## Goals

1. **Simplify**: Single base `docker-compose.yml` + 4 environment overrides
2. **Integration environment**: Local development against remote staging services via SSH tunnels + Traefik basic auth
3. **Clean secrets**: Per-environment secret folders with refactored setup script
4. **UX**: Makefile convenience commands (`make dev`, `make integration`, etc.)
5. **Cleanup**: Remove all legacy/archived compose files

## Architecture

### Approach: Base + Override (Classic Docker Pattern)

Single `docker-compose.yml` defines all services without ports, env_file, or environment variables. Each `compose.<env>.yml` override adds environment-specific configuration.

> **MIGRATION NOTE**: The existing `docker-compose.yml` will be **rewritten from scratch**. The current file contains inline `env_file`, `ports`, and `environment` blocks that violate the new base principles. All existing profile assignments (`minimal`, `dev`, `observability`, `full`) will be replaced with the new profile set defined below. The current file serves as reference for service definitions only.

## Base Compose Principles

- **No `env_file`** in base — each override loads from its secrets folder
- **No ports exposed** in base — each override decides binding (localhost vs 0.0.0.0)
- **Healthchecks** defined in base for all services
- **Network** `meepleai` (bridge) defined in base
- **Volumes** with neutral names (no environment suffix)
- **Build contexts** defined in base (Dockerfile paths)
- **`depends_on`** with `condition: service_healthy` in base

## Environment Details

### Dev (`compose.dev.yml`)

- **What runs**: Everything local — core + ai + monitoring + automation + proxy + storage
- **Ports**: All on `127.0.0.1`
- **Secrets**: Auto-generated by `setup-secrets.ps1 -Environment dev -SaveGenerated`
- **Volumes**: Suffixed `_dev_data` for isolation
- **Command**: `make dev` (all profiles) or `make dev-core` (core only)

### Integration (`compose.integration.yml`)

- **What runs locally**: Only `api` + `web`
- **Remote services**: Postgres, Redis, Qdrant via SSH tunnel; AI services via Traefik basic auth
- **Infrastructure services**: Simply not declared in `compose.integration.yml` — only `api` and `web` are defined. Since the base file has no `env_file` or `ports`, undeclared services have no configuration and won't interfere.
- **Ports**: `api` at `127.0.0.1:8080`, `web` at `127.0.0.1:3000` (must be explicitly declared in override)
- **Platform**: Requires `extra_hosts: ["host.docker.internal:host-gateway"]` on `api` and `web` services for Linux/WSL2 compatibility (Docker Desktop on Windows/macOS injects this automatically)
- **Connection pattern**:
  - DB/Redis/Qdrant → SSH tunnel → `host.docker.internal:15432/16379/16333`
  - AI services → `https://meepleai.app/services/embedding` (basic auth)
- **Command**: `make tunnel` then `make integration`

#### SSH Tunnel Configuration

| Service | Local Port | Remote Port | Host |
|---------|-----------|-------------|------|
| PostgreSQL | 15432 | 5432 | `host.docker.internal` |
| Redis | 16379 | 6379 | `host.docker.internal` |
| Qdrant | 16333 | 6333 | `host.docker.internal` |

Port offset +10000 to avoid conflicts with local dev.

#### SSH Tunnel Implementation (`ssh-tunnel.ps1`)

- **Single multiplexed SSH connection**: Uses `ControlMaster=auto` + `ControlPath=~/.ssh/meepleai-tunnel.sock` for a single SSH process forwarding all 3 ports
- **Background mode**: `ssh -fN` (fork to background, no remote command)
- **Auto-reconnect**: Not built-in — if connection drops, run `make tunnel` again. The script checks for existing tunnel first and skips if already active.
- **Stop mechanism**: `make tunnel-stop` sends SIGTERM to the SSH process identified by the control socket (`ssh -O exit -S ~/.ssh/meepleai-tunnel.sock`)
- **Status check**: `make tunnel-status` uses `ssh -O check -S ~/.ssh/meepleai-tunnel.sock` to verify tunnel health
- **Failsafe**: `ExitOnForwardFailure=yes` ensures SSH exits if any port forward fails (e.g., port already in use)

### Staging (`compose.staging.yml`)

- **Server**: `deploy@204.168.135.69` (Hetzner CAX21 ARM64)
- **Domain**: `meepleai.app`
- **What runs**: core + ai + monitoring + proxy
- **Ports**: None exposed directly — all via Traefik
- **TLS**: Let's Encrypt via HTTP challenge
- **Basic auth**: On `/services/*` paths for integration access
- **Volumes**: Retain existing volume names (`meepleai-pgdata-staging`, `meepleai-qdrantdata-staging`) to preserve live data. New services use `_staging` suffix convention.

> **VOLUME MIGRATION**: The staging server has live data in existing named volumes. The new compose files MUST use the same volume names to avoid data loss. Map in `compose.staging.yml` via `volumes: { postgres_data: { name: meepleai-pgdata-staging } }` pattern.

### Services Auth Credential Flow

**Staging side** (`traefik.secret`): Contains `INTEGRATION_BASIC_AUTH` in htpasswd format (`user:$apr1$...`), loaded by Traefik middleware on `/services/*` paths.

**Integration side** (`services-auth.secret`): Contains plaintext credentials matching the htpasswd:
```bash
SERVICES_AUTH_USER=integration
SERVICES_AUTH_PASS=<generated-password>
```
The `api` container reads these env vars and uses them as HTTP Basic Auth headers when calling `https://meepleai.app/services/*`. The `setup-secrets.ps1 -Environment integration` script generates both the plaintext and htpasswd versions.

### Key Differences

| Secret | Dev | Integration | Staging/Prod |
|--------|-----|-------------|--------------|
| `POSTGRES_HOST` | `postgres` | `host.docker.internal` | `postgres` |
| `POSTGRES_PORT` | `5432` | `15432` | `5432` |
| `EMBEDDING_URL` | `http://embedding:8000` | `https://meepleai.app/services/embedding` | `http://embedding:8000` |

### Setup Script (`setup-secrets.ps1`)

Refactored with `-Environment` parameter:

| Environment | `-SaveGenerated` | Behavior |
|-------------|------------------|----------|
| dev | Yes | Auto-generate random passwords, save immediately |
| dev | No | Generate and show, ask confirmation |
| integration | — | Prompt to copy from staging, configure tunnel ports |
| staging | — | Interactive prompt per secret, generate htpasswd |
| prod | — | Interactive + strict validation (min 16 chars) |

## Networking

- **Network**: `meepleai` (bridge), defined in base
- **Dev**: All ports on `127.0.0.1`
- **Integration**: No infra ports — SSH tunnels + Traefik
- **Staging/Prod**: No direct port exposure — all via Traefik
- **Docker socket proxy**: v0.6 (fix from current v0.2 API compatibility issue)

## Traefik Configuration

### Shared (`compose.traefik.yml`) — Linux only
- **Platform**: Linux only (staging/prod servers). Dev on Windows does not use Traefik — ports are exposed directly.
- Docker socket proxy v0.6 (Unix socket `/var/run/docker.sock`)
- Traefik v3.6 with Docker provider via socket proxy (`tcp://docker-socket-proxy:2375`)
- Cert volume for Let's Encrypt
- No Windows `npipe` support — if local Traefik is needed on Windows, use `compose.dev.yml` with direct port binding instead

### Staging (`traefik.staging.yml`)
- HTTP → HTTPS redirect
- Let's Encrypt ACME HTTP challenge
- Domain: `meepleai.app`
- Basic auth middleware for `/services/*`
- Rate limiting (100 avg, 50 burst)

## Quick Start Flows

### New Developer
```bash
cd infra && make secrets-dev && make dev
# → localhost:8080 (api), localhost:3000 (web)
```

### Test Against Real Services
```bash
cd infra && make secrets-integration && make tunnel && make integration
# → localhost:8080 (local api → remote DB/AI)
```

### Deploy Staging
```bash
ssh deploy@204.168.135.69
cd /opt/meepleai/repo/infra && make secrets-staging && make staging
# → https://meepleai.app
```

### Deploy Production (Future)
```bash
PROD_DOMAIN=meepleai.com make prod
```
