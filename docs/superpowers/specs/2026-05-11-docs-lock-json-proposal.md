# ADR Draft — docs.lock.json: file → SHA inventory for docs/

**Status**: Draft — proposal post #979 release-PR triage
**Date**: 2026-05-11
**Context**: 3 cleanup PRs (#1002, #1013) had to exclude 4 doc tree paths from Lychee because of cascade deadlinks introduced by reorgs (#916 Qdrant cleanup, #949 ALPHA_MODE removal). Tracking: #1014.

## Problem

When file A in `docs/` references file B, deletion or move of B silently leaves A with a dead link. Current CI gate (Lychee) catches this only on PR-touched files via diff — pre-existing dead links accumulate. Cross-tree refs (e.g., `for-claude/` referencing `for-developers/`) bypass diff filters entirely.

## Proposal

Add `docs/.lock.json` mapping every Markdown file path to its content SHA at commit-time, generated by pre-commit hook:

```json
{
  "version": 1,
  "files": {
    "docs/for-developers/deployment/r2-storage-configuration-guide.md": {
      "sha256": "abc123...",
      "referenced_by": [
        "docs/for-developers/operations/operations-manual.md",
        "docs/for-developers/deployment/README.md"
      ]
    },
    ...
  }
}
```

Hook generates `referenced_by` by parsing markdown `**text** _(planned)_` and `file://` links. Pre-commit fails if:
- File deleted but `referenced_by` non-empty in lock
- File moved (SHA same, path different) without updating refs

## Trade-offs

| Pro | Con |
|-----|-----|
| Pre-commit catches dead links shift-left | Lock file noise in diffs |
| Cross-tree refs tracked | Initial bootstrap cost (~scan all docs/) |
| Migration safety net | Hook adds ~2-5s commit latency |

## Alternatives considered

- **Pre-commit lychee --offline** (simpler) — see separate ADR. No cross-tree awareness but easier setup.
- **GitHub action only** (status quo + Lychee tightening) — but then debt accumulates on `main`.
- **Vale linter** (broader scope) — overshoots problem.
