# US-10: Wishlist Frontend Activation

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox syntax for tracking.

**Goal:** Activate the wishlist tab in the library section, allowing users to maintain a wishlist of games they want to acquire — with priority, target price, and notes.

**Architecture:** Frontend-only activation. The backend Wishlist API is complete (`/api/v1/wishlist` CRUD). The frontend has existing components (`MeepleWishlistCard`, `AddToWishlistDialog`), API client (`wishlistClient.ts`), React Query hooks (`useWishlist.ts`), and Zod schemas. The main work is: (1) activate the wishlist tab in alpha mode, (2) wire the `_content.tsx` tab router to render the wishlist page, (3) add "Add to Wishlist" actions on game cards, and (4) resolve the `MeepleWishlistCard` showing raw `gameId` instead of game name.

**Tech Stack:** Next.js 16, React 19, Tailwind 4, shadcn/ui

**Parent Branch:** `frontend-dev`
**Feature Branch:** `feature/us-10-wishlist`

## Task 4: Improve AddToWishlistDialog with Game Search

**Files:**
- Modify: `apps/web/src/components/wishlist/AddToWishlistDialog.tsx`

### Context
The current dialog requires typing a raw UUID for `gameId`. This is unusable. It should either:
- Accept a pre-filled `gameId` + `gameName` when triggered from a game card (already supports `gameId` prop)
- Provide a search input when opened standalone (from the wishlist page header)

### Steps

- [ ] Step 1: When `gameId` is NOT pre-filled, replace the raw UUID input with a search-based game selector. Use the existing `api.games.search()` or `api.games.getAll({ search })` pattern:
  ```typescript
  import { useDebounce } from '@/hooks/useDebounce'; // or implement inline
  import { api } from '@/lib/api';
  import { useQuery } from '@tanstack/react-query';

  // Inside the component, when gameId is not pre-filled:
  const [searchTerm, setSearchTerm] = useState('');
  const debouncedSearch = useDebounce(searchTerm, 300);

  const { data: searchResults } = useQuery({
    queryKey: ['games', 'search', debouncedSearch],
    queryFn: () => api.games.getAll({ search: debouncedSearch, pageSize: 10 }),
    enabled: debouncedSearch.length >= 2 && !gameId,
  });
  ```

- [ ] Step 2: Render a searchable dropdown/combobox that shows game results and sets `formGameId` + displays the selected game name. Use a simple list with click-to-select:
  ```tsx
  {!gameId && (
    <div className="space-y-1.5">
      <Label htmlFor="wishlist-game-search">Search Game</Label>
      <Input
        id="wishlist-game-search"
        placeholder="Type to search games..."
        value={searchTerm}
        onChange={e => setSearchTerm(e.target.value)}
      />
      {searchResults?.items && searchResults.items.length > 0 && (
        <ul className="max-h-40 overflow-y-auto border rounded-md divide-y">
          {searchResults.items.map(game => (
            <li
              key={game.id}
              className="px-3 py-2 hover:bg-accent cursor-pointer text-sm"
              onClick={() => {
                setFormGameId(game.id);
                setSelectedGameName(game.title);
                setSearchTerm('');
              }}
            >
              {game.title}
            </li>
          ))}
        </ul>
      )}
      {formGameId && selectedGameName && (
        <p className="text-sm text-muted-foreground">
          Selected: <strong>{selectedGameName}</strong>
        </p>
      )}
    </div>
  )}
  ```

- [ ] Step 3: When `gameId` IS pre-filled (opened from a game card), show the game name as a read-only field instead of UUID. Add an optional `gameName` prop:
  ```typescript
  interface AddToWishlistDialogProps {
    trigger?: React.ReactNode;
    gameId?: string;
    gameName?: string; // Display name when gameId is pre-filled
    onSuccess?: () => void;
  }
  ```
