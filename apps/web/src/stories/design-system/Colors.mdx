## Accessibility Notes

### WCAG 2.1 AA Compliance

All color combinations have been tested for minimum contrast ratios:

| Element | Requirement | Our Ratio |
|---------|-------------|-----------|
| Normal text | 4.5:1 | 4.8:1+ |
| Large text | 3:1 | 5.2:1+ |
| UI components | 3:1 | 4.1:1+ |

### Color Adjustments for Compliance

Several colors were darkened from their original values to meet WCAG requirements (Issue #2234):

- **Primary**: Darkened from 53% to 38% lightness
- **Secondary**: Darkened from 36% to 28% lightness
- **Accent**: Darkened from 65% to 55% lightness
- **Muted Foreground**: Darkened to 35% for guaranteed contrast

## Usage

### In Tailwind Classes

```tsx
// Semantic colors
<div className="bg-primary text-primary-foreground" />
<div className="bg-secondary text-secondary-foreground" />
<div className="bg-destructive text-destructive-foreground" />

// UI colors
<div className="bg-background text-foreground" />
<div className="bg-muted text-muted-foreground" />
<div className="bg-card text-card-foreground" />
```

### In CSS Variables

```css
.custom-element {
  background: hsl(var(--primary));
  color: hsl(var(--primary-foreground));
  border-color: hsl(var(--border));
}
```

### With Opacity

```css
/* HSL allows easy opacity adjustment */
.overlay {
  background: hsl(var(--primary) / 0.5);
}
```
