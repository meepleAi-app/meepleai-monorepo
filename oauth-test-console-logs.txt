=== ANALISI PROBLEMA OAUTH GOOGLE ===

FLUSSO:
1. User clicca "Continua con Google"
2. Redirect a Google OAuth → Login/Autorizzazione
3. Redirect callback a MeepleAI → /auth/callback/google (o simile)
4. La pagina OAuthCallbackPageContent si carica
5. Il componente tenta di navigare usando router.push
6. PROBLEMA: Nel frattempo AuthProvider e useSessionCheck tentano di verificare la sessione
7. Gli endpoint /auth/me e /auth/session/status restituiscono 401 perché la sessione non è ancora creata
8. Il flusso si blocca e torna alla pagina di login

ROOT CAUSE:
Il backend non completa correttamente la creazione della sessione dopo il callback OAuth di Google.
Gli hook React tentano di verificare la sessione prima che sia stata stabilita.

SOLUZIONE PROPOSTA:
- Verificare che il backend crei correttamente la sessione/cookie dopo il callback OAuth
- Aggiungere gestione errori nel componente OAuthCallbackPageContent
- Ritardare le chiamate a /auth/me fino a conferma sessione stabilita
- Aggiungere loading state durante il processo di callback

=== FIX APPLICATO ===

**File modificato**: apps/api/src/Api/appsettings.json

**Configurazione aggiunta** (line 66-73):
```json
"SessionCookie": {
  "Name": "meepleai_session",
  "Secure": false,
  "SameSite": "None",
  "HttpOnly": true,
  "Path": "/",
  "UseForwardedProto": true
}
```

**Rationale:**
- Secure: false → Permette cookie su HTTP localhost
- SameSite: None → Permette cross-port (3000→8080)
- HttpOnly: true → Protegge da XSS
- UseForwardedProto: true → Supporta proxy reversi

**NOTA SICUREZZA:**
Questa configurazione è SOLO per development locale.
In production DEVE essere:
- Secure: true (richiesto per HTTPS)
- SameSite: "Lax" o "Strict" (maggiore sicurezza)
