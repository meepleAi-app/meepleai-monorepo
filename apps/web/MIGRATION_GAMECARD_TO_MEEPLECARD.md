# GameCard → MeepleCard Migration Guide

**Epic #4029** | **Issue #4032**

## Overview

Legacy `GameCard` components are deprecated in favor of the unified `MeepleCard` component system with entity-specific quick actions.

**Benefits of MeepleCard**:
- ✅ Supports 7 entity types (not just games)
- ✅ Entity-specific quick actions with hover animations
- ✅ Info button navigation to detail pages
- ✅ Board game card proportions (7:10 aspect ratio)
- ✅ Modular feature system (wishlist, status, preview, drag, bulk select)
- ✅ Consistent design across all entity types

## Quick Migration

### Before (Legacy GameCard)

```tsx
import { GameCard } from '@/components/games/GameCard';

<GameCard
  data={game}
  viewMode="grid"
  onClick={() => router.push(`/games/${game.id}`)}
  onAskAI={() => openChat(game.id)}
/>
```

### After (MeepleCard)

```tsx
import { MeepleCard } from '@/components/ui/data-display';
import { useEntityActions } from '@/hooks/use-entity-actions';

const actions = useEntityActions({ entity: 'game', id: game.id });

<MeepleCard
  entity="game"
  variant="grid"
  title={game.name}
  subtitle={`${game.publisher}, ${game.yearPublished}`}
  imageUrl={game.imageUrl}
  rating={game.averageRating}
  ratingMax={10}
  metadata={[
    { icon: Users, value: `${game.minPlayers}-${game.maxPlayers}` },
    { icon: Clock, value: `${game.playingTime}m` },
  ]}
  entityQuickActions={actions.quickActions}
  showInfoButton
  infoHref={`/games/${game.id}`}
  onClick={() => router.push(`/games/${game.id}`)}
/>
```

## Prop Mapping Reference

| GameCard Prop | MeepleCard Prop | Notes |
|---------------|-----------------|-------|
| `data.name` | `title` | Direct mapping |
| `data.publisher` | Part of `subtitle` | Combine with year |
| `data.yearPublished` | Part of `subtitle` | Format: "Publisher, YYYY" |
| `data.imageUrl` | `imageUrl` | Direct mapping |
| `data.averageRating` | `rating` | Direct mapping |
| - | `ratingMax={10}` | BGG uses 0-10 scale |
| `data.minPlayers/maxPlayers` | `metadata[0]` | With Users icon |
| `data.playingTime` | `metadata[1]` | With Clock icon |
| `data.complexity` | `metadata[2]` | With BarChart icon |
| `viewMode` | `variant` | "grid" or "list" |
| `onClick` | `onClick` | Direct mapping |
| `onAskAI` | Via `entityQuickActions` | Use "Chat con Agent" action |

## Entity-Specific Actions

Quick actions are generated by `useEntityActions` hook:

```tsx
const actions = useEntityActions({
  entity: 'game',
  id: game.id,
  userId: currentUserId,
  userRole: currentUserRole,
});

// Returns:
// - Chat con Agent → /chat?gameId=[id]
// - Avvia Sessione → /sessions/new?gameId=[id]
// - Condividi → Copy link
```

## Grid Layout (Board Game Card Dimensions)

```tsx
<div className="grid grid-cols-[repeat(auto-fill,minmax(220px,240px))] gap-6">
  {games.map(game => (
    <MeepleCard
      key={game.id}
      entity="game"
      variant="grid"
      // ... props
    />
  ))}
</div>
```

Cards are 220-240px wide with 7:10 aspect ratio (like real board game cards).

## Feature Flags

Enable optional features:

```tsx
<MeepleCard
  // ... base props ...

  // Feature: Wishlist
  showWishlist
  isWishlisted={game.isWishlisted}
  onWishlistToggle={handleWishlist}

  // Feature: Status Badge
  status={game.status} // 'owned' | 'wishlisted' | 'played'

  // Feature: Hover Preview
  showPreview
  onFetchPreview={fetchGameDetails}

  // Feature: Drag & Drop
  draggable
  dragData={{ id: game.id, type: 'game', index: i }}

  // Feature: Bulk Selection
  selectable
  selected={selectedGames.includes(game.id)}
  onSelect={handleSelect}
/>
```

## Legacy Component Status

### Already Deprecated (Safe to Use)
- `components/games/GameCard.tsx` - Already wraps MeepleCard internally
- Multiple domain-specific wrappers exist

### Recommended for New Development
- ✅ **Always use** `MeepleCard` for new features
- ✅ **Use** `useEntityActions` for entity-specific actions
- ❌ **Avoid** creating new GameCard variants

## Migration Strategy (For Existing Code)

**Phase 1**: New features only (✅ Current)

**Phase 2**: Incremental refactor (Future)

- One domain at a time (games → library → dashboard → admin)
