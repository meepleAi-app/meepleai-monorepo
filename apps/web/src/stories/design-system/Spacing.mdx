# Spacing System

MeepleAI uses a **4px base unit** spacing system for consistent, harmonious layouts. All spacing values are multiples of 4px (0.25rem), creating visual rhythm throughout the interface.

## Base Scale

The spacing scale follows a predictable progression based on the 4px unit.

  --space-0: 0;
  --space-1: 0.25rem;   /* 4px */

## Semantic Spacing

For easier usage, we provide semantic aliases that map to common use cases.

| Token | Maps To | Pixels | Use Case |
|-------|---------|--------|----------|
| `--space-xs` | space-2 | 8px | Tight gaps, icon spacing |
| `--space-sm` | space-3 | 12px | Small gaps, inline elements |
| `--space-md` | space-4 | 16px | Default spacing, buttons |
| `--space-lg` | space-6 | 24px | Section spacing, cards |
| `--space-xl` | space-8 | 32px | Large gaps, section breaks |
| `--space-2xl` | space-12 | 48px | Major sections |
| `--space-3xl` | space-16 | 64px | Page-level spacing |

### Padding & Margin

```tsx

<div className="p-4">16px padding</div>
<div className="p-6">24px padding</div>
<div className="m-8">32px margin</div>

// Directional
<div className="px-4 py-2">Horizontal/vertical</div>

```

### Gap (Flexbox/Grid)

```tsx

```

## Utility Classes

Pre-defined spacing patterns for common layouts.

```css
/* Page-level spacing */
.space-page {
  padding: var(--space-6);   /* 24px */
}

/* Section spacing */
.space-section {
  padding: var(--space-8) var(--space-6);   /* 32px 24px */
}

/* Card spacing */
.space-card {
  padding: var(--card-padding-md);   /* 24px */
}
```

## Best Practices

1. **Use the scale** - Don't use arbitrary values like 7px or 13px
2. **Be consistent** - Same elements should have same spacing
3. **Increase with size** - Larger containers need more internal spacing
4. **Consider touch** - Mobile touch targets must be at least 44px
5. **Vertical rhythm** - Use consistent vertical spacing for readability
