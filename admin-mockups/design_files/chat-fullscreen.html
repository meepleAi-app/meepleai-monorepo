Pattern decisions consolidati

Riepilogo decisioni che differenziano la chat full-screen rispetto allo slide-over component-mock e ai pattern di reference.

🪟 Slide-over ↔ full-screen routing (viewport-driven):
Mobile/tablet (<1024px) → full-screen sempre (slide-over diventa drawer fallback)
Desktop (≥1024px) → slide-over default (`nanolith-nav-chat-panel.html`), toggle "↗ Apri full-screen" → questa route (`/chat/[threadId]`)
⚡ Streaming (riusa hook PR#551):
Token-by-token con AbortController via `useThreadMessages` hook. Cancel istantaneo <100ms.
Stati Nygard failure modes (tutti e 4 mostrati nel mockup): idle · streaming · stream-error · stream-abort.
Latency targets: first-token P50<500ms, P95<1500ms, idle 30s, hard timeout 60s.
📖 Citation pill (RIUSO `sp4-citation-pdf-viewer`):
Click pill → PDF viewer overlay (G4 v3 pattern, ownership-gated). NO fork.
Variant Aaron: pill `.cit-pill.translate` (color `--c-kb`) per citation da translate flow.
📷 Attachment Aaron-specific (ponte chat ↔ translate):
Tool 📎 in composer apre flow translate (`librogame-runthrough-translate-viewer.html` ?mode=manual o camera).
Risposta translate ritorna come citation pill `.cit-pill.translate` in chat (paragrafo §123 IT).
Voice 🎤 = UI placeholder (Whisper integration deferita v1).
📖 Reader mode + 🔆 wake-lock (coerenza Task 2 sez 1.1.1):
Toggle 📖 nell'header attiva font 24pt sui bubble agent (`localStorage 'reader-mode-enabled'`).
Stessa key del translate-viewer (Task 2 sez 1b.B) → cross-route consistency Aaron passive co-player.
Badge wake-lock 🔆 nel telemetry footer durante streaming (3 stati: attivo · disabilitato · non-supportato).
💸 Cost transparency (coerenza Task 1-4):
Telemetry footer mostra cost real-time (€0,xxxx) · tok consumati · quota residua · stato wake-lock.
stream-abort fattura solo token effettivi (124 tok = €0,0011, non query intera).
Cost-aware suggestions in empty state: cache hit paragrafi → 0 cost; Encounter re-OCR ~€0,01.
