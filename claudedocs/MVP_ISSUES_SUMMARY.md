## 🎓 Using the Issues

### Each Issue Contains

1. **Objective**: Clear goal statement
2. **Task List**: 5-10 checkable tasks
3. **Testing Requirements**: Coverage targets, test types
4. **Definition of Done**: Acceptance criteria
5. **References**: Links to specs and docs

### Example Workflow

```bash
# 1. Pick an issue
gh issue view 846

# 2. Create feature branch
git checkout -b feature/oauth-tests-846

# 3. Work on tasks (check them off in GitHub UI)
# - [ ] Task 1
# - [x] Task 1 (checked when done)

# 4. Commit with issue reference
git commit -m "feat(auth): add OAuth unit tests

Closes #846"

# 5. Create PR
gh pr create --title "OAuth Integration Tests" --body "Closes #846"

# 6. Merge PR → Issue auto-closes
```
