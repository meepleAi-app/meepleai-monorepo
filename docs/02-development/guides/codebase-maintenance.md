## Cache Cleanup (Monthly)

### Automated Script

**Command**:
```bash
# Bash (Linux/Mac)
bash tools/cleanup-caches.sh --dry-run  # Preview

# PowerShell (Windows)
pwsh tools/cleanup-caches.ps1 -DryRun  # Preview
```

### Directories Cleaned

| Directory | Purpose | Safe to Delete | Regenerated By |
|-----------|---------|----------------|----------------|
| `.serena/` | Serena MCP cache | ✅ Yes | Serena MCP (on demand) |

| `.playwright-mcp/` | Playwright MCP cache | ✅ Yes | Playwright MCP (on demand) |

| `apps/web/node_modules` | NPM packages | ⚠️ Caution | `pnpm install` (slow!) |

**Recommended**: Clean caches monthly, skip node_modules unless broken

## TODO/FIXME Comment Management

### Pattern: No Long-Lived TODOs in Production Code

**Policy**: TODOs allowed ONLY in:
- Active feature branches (short-lived, < 1 week)
- Documentation files (if tracked in issues)

**Enforcement**:
```bash
# Audit script (run monthly)
grep -r "TODO\|FIXME" src/ --exclude-dir=node_modules --exclude-dir=bin --exclude-dir=obj
```

**Action on Found TODOs**:
1. **If still relevant**: Create GitHub issue, link in comment
   ```csharp
   // TODO: Optimize query performance (#1234)
   var results = await _context.Users.ToListAsync();
   ```

2. **If completed**: Remove comment
3. **If obsolete**: Remove comment

---

### Acceptable TODO Pattern

```csharp
// ✅ GOOD: Linked to issue with context
// TODO(#1234): Refactor to use CQRS after GameManagement context created
var games = await _gameService.GetAllAsync();

// ✅ GOOD: Short-term with clear action
// TODO: Extract to separate method (before PR merge)
var result = ComplexLogic();

// ❌ BAD: Vague, no tracking
// TODO: fix this later
var data = GetData();

// ❌ BAD: Long-lived without issue
// TODO: Add caching (added 6 months ago, no issue)
var users = await _userRepo.GetAllAsync();
```

## Git Hygiene

### Branch Cleanup

**After PR Merged**:
```bash
# Delete local branch immediately
git branch -D feature/my-feature

# Prune remote tracking branches (monthly)
git remote prune origin

# List orphan branches (remote deleted)
git branch -vv | grep ": gone]"

# Delete all orphan branches
git branch -vv | grep ": gone]" | awk '{print $1}' | xargs git branch -D
```

**Automation** (.git/hooks/post-merge):
```bash
#!/bin/bash
# Auto-delete merged feature branches

BRANCH=$(git rev-parse --abbrev-ref HEAD)
if [ "$BRANCH" = "main" ]; then
  # Cleanup merged branches
  git branch --merged | grep -v "main\|master\|develop" | xargs git branch -d
fi
```

### Commit Message Quality

**Pattern** (Conventional Commits):

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Types**:
- `feat`: New feature
- `fix`: Bug fix
- `refactor`: Code refactoring (no behavior change)
- `test`: Test changes
- `docs`: Documentation
- `chore`: Maintenance (deps, cleanup)
- `perf`: Performance improvement
- `ci`: CI/CD changes

**Examples**:
```bash
# ✅ GOOD: Clear, scoped, descriptive
feat(auth): Add OAuth integration with Google/Discord/GitHub
fix(chat): Resolve streaming timeout after 30s
refactor(ddd): Create GameManagement bounded context
test(e2e): Increase coverage to 80%+ critical journeys
docs(kb): Extract To-Be patterns from closed issues

# ❌ BAD: Vague, no context
fix: bugs
update: stuff
wip: changes
```

**Enforcement**: Pre-commit hook (optional)

```bash
# .git/hooks/commit-msg
#!/bin/bash
commit_msg=$(cat "$1")
if ! echo "$commit_msg" | grep -qE "^(feat|fix|refactor|test|docs|chore|perf|ci)(\(.+\))?: .+"; then
  echo "ERROR: Commit message doesn't follow Conventional Commits format"
  echo "Format: <type>(<scope>): <subject>"
  exit 1
fi
```

## Documentation Hygiene

### Regular Documentation Cleanup (Quarterly)

**Process**:
1. **Identify Obsolete Docs**:
   ```bash
   # Find docs related to closed issues
   gh issue list --state closed --limit 100
   # Cross-reference with claudedocs/
   ```

2. **Extract To-Be Knowledge**:
   - Best practices → kb/
   - Patterns to reuse → kb/
   - Architectural decisions → docs/architecture/

3. **Remove Obsolete**:
   ```bash

   mkdir -p claudedocs/archive/$(date +%Y-%m)
   mv claudedocs/obsolete-*.md claudedocs/archive/2025-11/
   ```

4. **Update Index**:
   - Update CLAUDE.md references
   - Update kb/README.md (if exists)

## Test Maintenance

### Test Suite Health (Monthly)

**Metrics to Track**:
```bash
# Frontend coverage
pnpm test:coverage
# Check: Statements 90%+, Branches 84%+

# Backend coverage
dotnet test /p:CollectCoverage=true
# Check: Line 90%+, Branch 85%+

# E2E coverage
pnpm test:e2e
# Check: Critical journeys 80%+

# Flaky tests
# Run test suite 5 times, any failures = flaky
for i in {1..5}; do pnpm test; done
```

**Action on Coverage Drop**:
- Below 85%: Add tests for uncovered code
- Below 80%: Block PRs until fixed
- Below 75%: Critical issue, immediate action

---

### Flaky Test Elimination

**Identify Flaky Tests**:
```bash
# Run suite multiple times
for i in {1..10}; do
  pnpm test 2>&1 | tee test-run-$i.log
done

# Grep for failures
grep "FAIL" test-run-*.log | sort | uniq -c
```

**Common Causes**:
1. Race conditions (async timing)
2. Shared state (test pollution)
3. External dependencies (API flakiness)
4. Non-deterministic data (random, dates)

**Fixes**:
```typescript
// ❌ FLAKY: Timing-dependent
test('Data loads', async () => {
  fetchData();
  await wait(1000); // Race condition!
  expect(data).toBeTruthy();
});

// ✅ STABLE: Wait for condition
test('Data loads', async () => {
  fetchData();
  await waitFor(() => expect(data).toBeTruthy(), { timeout: 5000 });
});
```

**Policy**: Zero tolerance for flaky tests (fix OR skip with .skip + issue)

### Creating Maintenance Scripts

**Best Practices**:

```bash
#!/bin/bash
# Script template

set -e  # Exit on error

# 1. Dry run mode
DRY_RUN=false
if [[ "$@" == *"--dry-run"* ]]; then
  DRY_RUN=true
fi

# 2. Confirmation prompt
if [ "$DRY_RUN" = false ]; then
  read -p "This will delete X. Continue? (y/N) " -n 1 -r
  echo
  if [[ ! $REPLY =~ ^[Yy]$ ]]; then
    exit 1
  fi
fi

# 3. Safe deletion (move to trash OR backup first)
if [ "$DRY_RUN" = true ]; then
  echo "[DRY RUN] Would delete: X"
else
  echo "Deleting X..."
  rm -rf X
fi

# 4. Summary
echo "Cleanup complete! Freed: XYZ MB"
```

**Features**:
- Dry run mode (--dry-run)
- Confirmation prompt (safety)
- Colored output (readability)
- Summary statistics (space saved)

## Git Repository Maintenance

### Large Files Audit

**Find Large Files**:
```bash
# Files > 1MB in current tree
find . -type f -size +1M -not -path "./.git/*" -not -path "*/node_modules/*"

# Large files in git history (bloat repo)
git rev-list --objects --all | \
  git cat-file --batch-check='%(objecttype) %(objectname) %(objectsize) %(rest)' | \
  awk '/^blob/ {if ($3 > 1048576) print $3/1048576 " MB", $4}' | \
  sort -rn | head -20
```

**Prevention** (.gitignore):
```
# Build artifacts
bin/
obj/
.next/
node_modules/

# Caches
.serena/
codeql-db/
.playwright-mcp/

# Large files
*.mp4
*.mov
*.zip
*.tar.gz

# Secrets (critical!)
.env.local
.env.dev
*.key
*.pem
```

**Pre-commit Hook** (prevent large files):
```bash
#!/bin/bash
# .git/hooks/pre-commit

# Check for large files
large_files=$(git diff --cached --name-only | xargs ls -l 2>/dev/null | awk '$5 > 1048576 {print $9}')

if [ -n "$large_files" ]; then
  echo "ERROR: Large files detected (>1MB):"
  echo "$large_files"
  echo ""
  echo "Use Git LFS for large files OR add to .gitignore"
  exit 1
fi
```

## Emergency Maintenance Procedures

### Critical Security Vulnerability

**Timeline**: Fix within 24 hours

**Process**:
1. Assess severity (CVSS score, exploit available?)
2. Check for patched version (dependency update)
3. If no patch: Implement workaround OR disable feature
4. Test fix thoroughly
5. Deploy hotfix (bypass normal release cycle)
6. Document in security incident log
7. Notify stakeholders

---

### Production Outage

**Timeline**: Restore service ASAP

**Process**:
1. Rollback to previous version (fastest)
2. Identify root cause (logs, monitoring)
3. Create hotfix branch
4. Fix + test
5. Deploy
6. Post-mortem (document lessons learned)
