Riepilogo comparativo
Criterio A — Inline B — Toolbar C — Glass Pill D — Sidebar
Azioni visibili scrollando ✗ No ✓ Sì ✓ Sì ✓ Sì
Spazio contenuto ✓ Massimo ✗ Ridotto (3 barre) ✓ Buono ~ Ridotto orizzontale
Continuità con mobile ~ Parziale ~ Parziale ✓ Alta (stessa metafora) ✗ Bassa
Consistenza tra pagine ✗ Dipende dal layout ✓ Alta ✓ Alta ✓ Alta
Familiare agli utenti ✓ Molto (GitHub/Notion) ✓ Sì ~ Nuovo ma intuitivo ✓ (VS Code/Figma)
Adatto a sessione attiva ✗ Azioni spariscono ✓ Sì ✓ Sì ✓ Sì
Implementazione ✓ Semplice ~ Media ~ Media ~ Media

Note: Il mockup usa la pagina Game Detail come test case perché ha il set di azioni più ricco. Pagine semplici (es. Lista libreria) funzionano bene con qualsiasi opzione. Le pagine con contenuto lungo (es. Sessione attiva) premiano le opzioni B/C/D dove le azioni rimangono visibili.
