# Backend Code Coverage Guide

This guide explains how to run code coverage analysis for the MeepleAI backend (.NET/C#).

## Quick Start

### Option 1: Local Execution (Recommended)

If you have .NET SDK 9.0 installed locally:

```bash
# Run tests with coverage and generate HTML report
./tools/run-backend-coverage.sh --html --open

# Or just run coverage without HTML report
./tools/run-backend-coverage.sh

# Set custom threshold
./tools/run-backend-coverage.sh --threshold 85
```

### Option 2: Docker-based Execution

If you don't have .NET SDK installed but have Docker:

```bash
# Run tests with coverage using Docker
./tools/run-backend-coverage-docker.sh

# Set custom threshold
./tools/run-backend-coverage-docker.sh --threshold 85
```

### Option 3: Manual Execution

```bash
cd apps/api

# Run tests with coverage collection
dotnet test \
  --logger "console;verbosity=normal" \
  -p:CollectCoverage=true \
  -p:CoverletOutputFormat=cobertura \
  -p:CoverletOutput=./tests/Api.Tests/coverage/ \
  -p:Threshold=90 \
  -p:ThresholdType=line \
  -p:Exclude="[*]Api.Migrations.*"

# Generate HTML report (requires reportgenerator)
dotnet tool install -g dotnet-reportgenerator-globaltool
reportgenerator \
  -reports:tests/Api.Tests/coverage/coverage.cobertura.xml \
  -targetdir:tests/Api.Tests/coverage-report \
  -reporttypes:Html

# Open the report
open tests/Api.Tests/coverage-report/index.html  # macOS
xdg-open tests/Api.Tests/coverage-report/index.html  # Linux
start tests/Api.Tests/coverage-report/index.html  # Windows
```

## Coverage Output

### Console Output

The test run will display:
- Test execution progress
- Test results (passed/failed)
- Coverage percentage
- Threshold validation

Example output:
```
==================================================
  Coverage Summary
==================================================
Line Coverage: 92%
Threshold:     90%
==================================================
```

### Coverage Files

| File | Location | Format | Purpose |
|------|----------|--------|---------|
| Cobertura XML | `apps/api/tests/Api.Tests/coverage/coverage.cobertura.xml` | XML | Machine-readable coverage data |
| HTML Report | `apps/api/tests/Api.Tests/coverage-report/index.html` | HTML | Human-readable coverage report |

### HTML Report Features

The HTML report generated by ReportGenerator includes:

- **Summary Dashboard**: Overall coverage metrics
- **File Explorer**: Navigate coverage by namespace/class
- **Line Coverage**: See which lines are covered/uncovered
- **Branch Coverage**: See which branches are covered
- **Metrics**: Cyclomatic complexity, coverage percentages
- **History**: Track coverage trends over time (if run multiple times)
- **Risk Hotspots**: Identify files with low coverage

## Coverage Thresholds

Current project thresholds:

| Metric | Threshold | Status |
|--------|-----------|--------|
| Line Coverage | ≥90% | ✅ Enforced |
| Branch Coverage | - | ❌ Not enforced (tracked only) |

### Threshold Enforcement

- **Local Development**: Recommended but not enforced
- **CI/CD**: Strictly enforced (build fails if below threshold)
- **Pull Requests**: Coverage must not decrease

## CI/CD Integration

Code coverage is automatically run in GitHub Actions CI pipeline:

```yaml
# .github/workflows/ci.yml
- name: Unit & Integration Tests with Coverage (Target >=90%)
  run: |
    dotnet test \
      -p:CollectCoverage=true \
      -p:CoverletOutputFormat=cobertura \
      -p:Threshold=90 \
      -p:ThresholdType=line
```

### Coverage Reports in CI

1. **Codecov**: Automatic upload to Codecov for tracking and PR comments
2. **Artifacts**: HTML reports are uploaded as artifacts on test failures
3. **PR Comments**: Codecov bot posts coverage diff in pull requests

View CI coverage reports:
- **Codecov Dashboard**: https://codecov.io/gh/DegrassiAaron/meepleai-monorepo
- **CI Artifacts**: Available in failed workflow runs (7-day retention)

## Understanding Coverage Metrics

### Line Coverage

Percentage of code lines executed during tests.

```csharp
// Example: 2/3 lines covered (66.67%)
public int Divide(int a, int b)
{
    if (b == 0) throw new ArgumentException("Cannot divide by zero");  // ✅ Covered
    return a / b;  // ❌ Not covered (test only checks exception case)
}
```

### Branch Coverage

Percentage of decision branches executed.

```csharp
// Example: 2/3 branches covered (66.67%)
public string GetGrade(int score)
{
    if (score >= 90) return "A";  // ✅ Covered
    if (score >= 80) return "B";  // ✅ Covered
    return "C";                   // ❌ Not covered
}
```

### Coverage Exclusions

The following are excluded from coverage analysis:

- **Migrations**: `[*]Api.Migrations.*`
- **Auto-generated code**: Files with `[ExcludeFromCodeCoverage]` attribute
- **Program.cs**: Application entry point (integration tested separately)

## Best Practices

### 1. Focus on Business Logic

Prioritize testing and coverage for:
- ✅ Domain models and aggregates
- ✅ Application handlers (Commands/Queries)
- ✅ Domain services
- ✅ Complex business rules

Lower priority:
- ⚠️ Infrastructure code (often integration tested)
- ⚠️ DTOs and data models (simple properties)
- ⚠️ Configuration classes

### 2. Test Quality Over Quantity

High coverage doesn't guarantee quality:

```csharp
// ❌ Bad: High coverage, low quality
[Fact]
public void Test_ProcessOrder_DoesNotThrow()
{
    var order = new Order();
    var exception = Record.Exception(() => orderService.Process(order));
    Assert.Null(exception);  // Covers code but doesn't verify behavior
}

// ✅ Good: Verifies actual behavior
[Fact]
public void ProcessOrder_ValidOrder_UpdatesStatus()
{
    // Arrange
    var order = new Order { Status = OrderStatus.Pending };

    // Act
    orderService.Process(order);

    // Assert
    Assert.Equal(OrderStatus.Processed, order.Status);
    Assert.NotNull(order.ProcessedAt);
}
```

### 3. Use Coverage to Find Gaps

Use coverage reports to identify:
- ❌ Untested edge cases
- ❌ Missing error handling tests
- ❌ Uncovered validation logic
- ❌ Untested exception paths

### 4. Maintain Coverage Over Time

- ✅ Run coverage locally before committing
- ✅ Review coverage reports in PRs
- ✅ Don't decrease coverage with new code
- ✅ Aim for 90%+ on new code

## Troubleshooting

### Coverage Below Threshold

**Problem**: Tests pass but coverage is below threshold

**Solutions**:
1. Check which files have low coverage:
   ```bash
   # View coverage report
   ./tools/run-backend-coverage.sh --html --open
   ```

2. Add tests for uncovered code:
   - Focus on Domain and Application layers first
   - Ensure edge cases are tested
   - Test error handling paths

3. Consider if code is testable:
   - Is the code too tightly coupled?
   - Can it be refactored for better testability?

### Tests Hanging

**Problem**: Tests hang during execution

**Solutions**:
1. Use blame-hang timeout (already configured):
   ```bash
   dotnet test --blame-hang-timeout 5m
   ```

2. Check for:
   - Deadlocks in async code
   - Infinite loops
   - Services waiting for external resources

### Docker Script Fails

**Problem**: Docker-based script fails to run

**Solutions**:
1. Ensure Docker services are running:
   ```bash
   cd infra && docker compose up -d meepleai-postgres meepleai-qdrant meepleai-redis
   ```

2. Check service health:
   ```bash
   docker compose ps
   curl http://localhost:6333/healthz  # Qdrant
   docker compose exec meepleai-redis redis-cli ping  # Redis
   docker compose exec meepleai-postgres pg_isready -U meeple  # PostgreSQL
   ```

3. Reset services if needed:
   ```bash
   docker compose down -v
   docker compose up -d meepleai-postgres meepleai-qdrant meepleai-redis
   ```

### ReportGenerator Not Found

**Problem**: HTML report generation fails

**Solution**: Install ReportGenerator globally:
```bash
dotnet tool install -g dotnet-reportgenerator-globaltool
```

Or use the project script which auto-installs it:
```bash
./tools/run-backend-coverage.sh --html
```

## Coverage Tools

### Coverlet

Primary coverage collection tool for .NET.

**Features**:
- ✅ Cross-platform (Windows, Linux, macOS)
- ✅ Multiple output formats (Cobertura, OpenCover, JSON, etc.)
- ✅ MSBuild integration
- ✅ Threshold enforcement

**Configuration**: `apps/api/Directory.Build.props`

```xml
<PropertyGroup>
  <CollectCoverage>true</CollectCoverage>
  <CoverletOutputFormat>cobertura</CoverletOutputFormat>
  <Threshold>90</Threshold>
  <ThresholdType>line</ThresholdType>
</PropertyGroup>
```

### ReportGenerator

Converts coverage data to human-readable reports.

**Supported Formats**:
- HTML (recommended)
- Markdown
- Badges
- SonarQube
- TeamCity
- Clover

**Documentation**: https://github.com/danielpalme/ReportGenerator

### Codecov

Cloud service for coverage tracking and visualization.

**Features**:
- 📊 Coverage dashboards
- 📈 Trend analysis
- 💬 PR comments with coverage diff
- 🚨 Alerts for coverage drops
- 🎯 File-level coverage tracking

**Dashboard**: https://codecov.io/gh/DegrassiAaron/meepleai-monorepo

## Advanced Usage

### Coverage for Specific Tests

```bash
# Run coverage for integration tests only
dotnet test \
  --filter "Category=Integration" \
  -p:CollectCoverage=true \
  -p:CoverletOutputFormat=cobertura

# Run coverage for specific namespace
dotnet test \
  --filter "FullyQualifiedName~KnowledgeBase" \
  -p:CollectCoverage=true
```

### Combine Multiple Coverage Reports

```bash
# Run unit tests
dotnet test tests/Api.Tests \
  -p:CollectCoverage=true \
  -p:CoverletOutput=./coverage/unit/ \
  -p:MergeWith=./coverage/coverage.json

# Run integration tests
dotnet test tests/Api.IntegrationTests \
  -p:CollectCoverage=true \
  -p:CoverletOutput=./coverage/integration/ \
  -p:MergeWith=./coverage/unit/coverage.json

# Generate combined report
reportgenerator \
  -reports:./coverage/**/*.xml \
  -targetdir:./coverage-report
```

### Coverage in Watch Mode

```bash
# Run tests in watch mode with live coverage updates
dotnet watch test \
  -p:CollectCoverage=true \
  -p:CoverletOutputFormat=cobertura
```
